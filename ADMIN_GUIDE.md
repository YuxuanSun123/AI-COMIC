# 后台管理系统说明文档

## 概述

后台管理系统是AICM Workshop的配置中心，提供API配置、系统设置、数据管理等功能。

## 访问方式

### 入口位置
- **URL**: `/admin` 或 `#/admin`
- **导航栏**: 登录后，顶部导航栏会显示"后台管理"入口
- **权限**: 需要登录才能访问

### 访问步骤
1. 注册/登录账号
2. 点击顶部导航栏的"后台管理"
3. 进入后台管理页面

## 功能模块

### 1. API配置

#### 1.1 AI生成API配置

**用途**: 配置OpenAI或兼容API，用于真实的AI剧本生成功能

**配置项**:

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 启用AI生成 | 开启后使用真实API，关闭则使用本地模拟 | 开关 |
| OpenAI API Key | API密钥 | sk-... |
| API Base URL | API端点地址 | https://api.openai.com/v1 |
| 模型名称 | 使用的模型 | gpt-4 |

**功能开关**:
- 剧本生成
- 分镜生成
- 镜头卡生成
- 剪辑计划生成

**操作**:
- 保存配置：保存到 localStorage
- 测试连接：测试API连接是否正常

#### 1.2 支持的API类型

1. **OpenAI官方API**
   - Base URL: `https://api.openai.com/v1`
   - 需要OpenAI API Key

2. **Azure OpenAI**
   - Base URL: `https://your-resource.openai.azure.com/`
   - 需要Azure API Key

3. **本地部署/第三方兼容API**
   - 支持任何OpenAI兼容的API端点
   - 如：LocalAI、FastChat等

### 2. 系统设置

#### 2.1 基本信息

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 网站名称 | 显示在页面标题和导航栏 | AI漫剧工作坊 |
| 网站描述 | 网站简介 | 专业的漫剧创作工具平台 |
| 每用户最大作品数 | 限制用户可创建的作品数量 | 100 |

#### 2.2 功能开关

| 功能 | 说明 | 默认状态 |
|------|------|----------|
| 允许用户注册 | 关闭后新用户无法注册 | 开启 |
| 邮箱验证 | 注册时是否需要验证邮箱（演示功能） | 关闭 |

#### 2.3 数据存储模式

**本地存储 (localStorage)**:
- 数据存储在浏览器本地
- 无需后端服务器
- 适合演示和原型验证

**远程API**:
- 数据存储在远程服务器
- 需要配置后端API地址
- 适合生产环境

#### 3.1 数据统计

实时显示本地存储的数据量：
- 用户数

#### 3.2 数据操作

**导出所有数据**:

**清空所有数据**:
- 清空localStorage中的所有数据
- 需要确认操作
- ⚠️ 此操作不可逆

## 配置存储

### 存储位置

所有配置保存在浏览器的 localStorage 中：

| 键名 | 内容 | 格式 |
|------|------|------|
| `admin_api_config` | API配置 | JSON |
| `admin_system_config` | 系统设置 | JSON |

### 配置结构

#### API配置 (admin_api_config)

```json
{
  "openai_api_key": "sk-...",
  "openai_base_url": "https://api.openai.com/v1",
  "openai_model": "gpt-4",
  "enable_ai_generation": false,
  "script_generation_enabled": true,
  "storyboard_generation_enabled": true,
  "video_cards_generation_enabled": true,
  "edit_plan_generation_enabled": true
}
```

#### 系统配置 (admin_system_config)

```json
{
  "site_name": "AI漫剧工作坊",
  "site_description": "专业的漫剧创作工具平台",
  "max_works_per_user": 100,
  "enable_registration": true,
  "enable_email_verification": false,
  "storage_mode": "local",
  "remote_api_url": ""
}
```

## 使用场景

### 场景1：配置OpenAI API进行真实生成

1. 进入后台管理 → API配置
2. 输入OpenAI API Key
3. 开启"启用AI生成"开关
4. 点击"测试连接"验证
5. 点击"保存配置"
6. 前往剧本生成器，生成内容将使用真实API

### 场景2：切换到本地模拟模式

1. 进入后台管理 → API配置
2. 关闭"启用AI生成"开关
3. 点击"保存配置"
4. 前往剧本生成器，生成内容将使用本地模拟

### 场景3：导出数据备份

1. 进入后台管理 → 数据管理
2. 点击"导出所有数据"
3. 下载JSON文件
4. 保存备份文件

### 场景4：配置Azure OpenAI

1. 进入后台管理 → API配置
2. 输入Azure API Key
3. 修改API Base URL为Azure端点
4. 修改模型名称为Azure部署名称
5. 开启"启用AI生成"
6. 测试并保存

## 安全说明

### ⚠️ 重要提示

1. **API Key安全**:
   - API Key存储在浏览器localStorage中
   - 仅演示用途，不适合生产环境
   - 生产环境应使用后端代理

2. **数据安全**:
   - 所有数据存储在浏览器本地
   - 清除浏览器数据会丢失所有配置
   - 建议定期导出备份

3. **权限控制**:
   - 当前版本仅检查登录状态
   - 未实现真实的管理员权限
   - 所有登录用户都可访问后台

## 常见问题

### Q1: 为什么配置了API Key还是使用本地模拟？

A: 需要开启"启用AI生成"开关，并保存配置。

### Q2: API Key会被泄露吗？

A: API Key存储在浏览器localStorage中，仅本地可见。但不建议在生产环境中使用此方式。

### Q3: 清空数据后能恢复吗？

A: 不能。清空操作会永久删除localStorage中的所有数据，建议先导出备份。

### Q4: 支持哪些AI模型？

A: 支持所有OpenAI兼容的API，包括：
- OpenAI官方模型（gpt-4, gpt-3.5-turbo等）
- Azure OpenAI
- 本地部署的兼容模型

### Q5: 如何切换到远程API存储？

A: 进入系统设置 → 数据存储模式 → 选择"远程API" → 配置API地址 → 保存。
